Semantic Matching Rules

  1. Exact IDs win: `future-003`, issue links, or `Closes #123` can mark an item claimed.
  2. Semantic matches are candidates only when title/body similarity is high enough.
  3. Low-confidence matches enter a review queue and do not change claim status.
  4. Maintainer confirmation writes the stable ID back into the issue or PR.
